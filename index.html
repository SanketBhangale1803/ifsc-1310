    Thoughts On Progressive Enhancement

    Progressive Enhancement

    Web design methodology that ensures that the content reaches all users first, regardless of their browser features and internet connection or simply digital advantage. "Graceful Degradation": It's opposite to Progressive Enhancement. This is the way of developing in which content is first available to users who have advanced browser versions and then degrade to work well with older browser versions.

    Core Principles Of Progressive Enhancement

    • All web browsers should be able to access basic content[global web standards]
    • Basic functions and standard events should be available in all web browsers. It's important to use normal DOM before implementing custom events to handle advanced functionalities.
    • All content should be contained in minimal,semantic markup.This enhances user-experience
    • Externally connected JavaScript should provide enhanced functionality
    • CSS that is externally connected should provide improved layout
    • End-user web browser preferences should be respected. User should be able to use whichever browser they wish, and should not be compelled to use a certain browser in order to access website content

    Why you should apply progressive enhancement to your web app?

    When it comes to online content delivery, there are a variety of challenges that might arise that can be caused by either the user or the developer. Progressive enhancement is a solution that was created to mitigate the consequences of these issues, or at the very least provide the end-user with some, if not all, information.
